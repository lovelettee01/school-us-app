# Phase 8 - 성능 점검 체크리스트

작성일: 2026-03-26
목표: 체감 성능, 네트워크 효율, 렌더링 안정성을 점검한다.

## 1) 핵심 지표 목표(초안)

- [ ] 초기 화면 First Contentful Paint 2.0s 이하
- [ ] 검색 결과 체감 응답 2.0s 이하(일반망)
- [ ] 탭 전환 시 주요 콘텐츠 표시 1.0s 이하
- [ ] 불필요한 중복 API 호출 0건

## 2) 네트워크 최적화

- [ ] 동일 파라미터 요청 캐시 동작 확인
- [ ] 탭 전환 시 이전 데이터 재사용 확인
- [ ] 연속 조회 시 이전 요청 취소(Abort) 확인
- [ ] API 오류 재시도 횟수 제한 확인

## 3) 렌더링 최적화

- [ ] 리스트/표 key 안정성 확인
- [ ] 무거운 컴포넌트(지도) 지연 로딩 적용 확인
- [ ] 테마 전환 시 레이아웃 점프 최소화
- [ ] 스켈레톤 표시 기준(짧은 지연 깜빡임 방지) 확인

## 4) 페이지별 점검

- 홈(`/`)
- [ ] 검색 폼 입력 지연 없음
- [ ] 결과 리스트 스크롤 성능 정상

- 상세(`/school/[schoolKey]`)
- [ ] 대표정보 로딩 후 탭 사용 가능
- [ ] 지도 탭 진입 시 UI 멈춤 없음
- [ ] 급식/시간표 탭 조회 후 재진입 성능 정상

## 5) 번들/리소스 점검

- [ ] 초기 번들 크기 과대 여부 점검
- [ ] 지도 SDK는 탭1에서만 로드되는지 확인
- [ ] 불필요한 폴리필/라이브러리 포함 여부 점검

## 6) 장애 상황 성능

- [ ] API 지연(3초+) 시 사용자 피드백 즉시 노출
- [ ] 네트워크 실패 후 재시도 흐름 지연 최소화
- [ ] 빈값 응답에서 과도한 리렌더 없음

## 7) 완료 기준

- [ ] P0 플로우(검색->상세->탭 전환) 성능 목표 충족
- [ ] 성능 회귀 감지 기준(기준치 +10% 초과) 정의
- [ ] 배포 전 성능 결과 기록 문서화
