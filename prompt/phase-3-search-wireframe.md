# Phase 3 - 검색 화면 와이어프레임

작성일: 2026-03-26
목표: 첫 화면(`/`)의 정보 구조와 주요 컴포넌트 배치를 확정한다.

## 1) 데스크톱 와이어프레임(텍스트)

```text
+----------------------------------------------------------------------------------+
| [Logo] 학교정보 조회                                         [Theme Toggle]       |
+----------------------------------------------------------------------------------+
| [시도교육청 Select ▼] [학교명 입력__________________] [조회 버튼] [초기화]      |
|  - 유효성 오류 메시지 영역 (필요 시 노출)                                         |
+----------------------------------------------------------------------------------+
| [필터/보조 영역]                                                                   |
|  - 최근 조회(가로 카드)                                                            |
|  - 즐겨찾기(가로 카드)                                                             |
+----------------------------------------------------------------------------------+
| 검색 결과                                                                          |
|  총 n건                                                                            |
|  +---------------------------------+ +-----------------------------------------+ |
|  | 학교명 / 학교급 / 설립          | | 주소, 연락처, 버튼(상세 보기, 즐겨찾기) | |
|  +---------------------------------+ +-----------------------------------------+ |
|  ...                                                                               |
|  [페이지네이션 또는 더보기]                                                        |
+----------------------------------------------------------------------------------+
| 상태 영역: 로딩/빈값/오류                                                         |
+----------------------------------------------------------------------------------+
```

## 2) 모바일 와이어프레임(텍스트)

```text
+--------------------------------------+
| 학교정보 조회            [Theme]      |
+--------------------------------------+
| 시도교육청 Select                      |
| 학교명 입력                            |
| [조회] [초기화]                        |
| (오류 메시지)                          |
+--------------------------------------+
| 최근 조회 (가로 스크롤 카드)           |
| 즐겨찾기 (가로 스크롤 카드)            |
+--------------------------------------+
| 검색 결과 총 n건                       |
| [학교 카드 1]                          |
| [학교 카드 2]                          |
| ...                                    |
| [더보기]                               |
+--------------------------------------+
| 로딩/빈값/오류 상태 영역               |
+--------------------------------------+
```

## 3) 컴포넌트 배치 규칙

- 헤더
- 로고/타이틀, 다크모드 토글 고정

- 검색 폼
- `시도교육청 선택`, `학교명 입력`, `조회`, `초기화`
- 엔터 입력 시 조회 트리거

- 보조 섹션
- 최근 조회, 즐겨찾기(선택 진입을 빠르게)

- 결과 섹션
- 카드 리스트 + 상세 진입 버튼 + 즐겨찾기 토글
- 리스트 항목 클릭 시 상세 페이지 이동

## 4) 시각적 상태 정의

- 로딩
- 리스트 스켈레톤 5개 + 버튼 비활성

- 빈값
- "조건에 맞는 학교가 없습니다" + 조건 변경 유도

- 오류
- 오류 메시지 + 재시도 버튼

## 5) 반응형 규칙

- 1024px 이상: 2열 기반 카드 정보 배치
- 768px~1023px: 1열 카드 + 메타 정보 축약
- 767px 이하: 세로 스택 + 터치 친화 버튼 크기(최소 44px)
