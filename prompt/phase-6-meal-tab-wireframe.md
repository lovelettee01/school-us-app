# Phase 6 - 급식 탭 와이어프레임

작성일: 2026-03-26
목표: 상세 페이지 탭 2(급식)의 UI 구성과 상태 표현을 확정한다.

## 1) 데스크톱 와이어프레임(텍스트)

```text
+----------------------------------------------------------------------------------+
| [탭] 학교정보/위치 | [탭] 급식(활성) | [탭] 시간표                                  |
+----------------------------------------------------------------------------------+
| [조회 기준: 오늘] [기간 선택(옵션)] [조회 버튼] [초기화]                         |
| 안내: 식단은 학교/날짜 조건에 따라 제공됩니다.                                   |
+----------------------------------------------------------------------------------+
| [요약 패널]                                                                       |
|  - 학교명                                                                          |
|  - 조회 기간                                                                       |
|  - 총 식단 건수                                                                    |
+----------------------------------------------------------------------------------+
| [일자 카드] 2026-03-26 (목) / 중식                                                |
|  메뉴: 쌀밥, 미역국, ...                                                           |
|  열량: 812.5kcal                                                                   |
|  알레르기: 1,5,6                                                                   |
+----------------------------------------------------------------------------------+
| [일자 카드] 2026-03-27 (금) / 중식                                                |
|  ...                                                                               |
+----------------------------------------------------------------------------------+
| 상태 영역: 로딩 스켈레톤 / 빈값 / 오류 + 재시도                                  |
+----------------------------------------------------------------------------------+
```

## 2) 모바일 와이어프레임(텍스트)

```text
+--------------------------------------+
| [탭] 정보 | [급식*] | [시간표]        |
+--------------------------------------+
| 조회 기준: 오늘                      |
| [날짜/기간 선택] [조회]              |
+--------------------------------------+
| 학교명 / 조회기간 / 총 건수          |
+--------------------------------------+
| [식단 카드 - 날짜]                   |
| 메뉴(줄바꿈)                         |
| 열량 / 알레르기                      |
+--------------------------------------+
| [식단 카드 - 날짜]                   |
+--------------------------------------+
| 로딩/빈값/오류 상태                   |
+--------------------------------------+
```

## 3) 표시 방식 결정

- 기본: 일별 카드 리스트
- 보조: 주간 범위 조회(선택)
- 정렬: 날짜 오름차순
- 오늘 날짜에 `TODAY` 뱃지 표시

## 4) 상호작용 규칙

- 기본 진입 시 오늘 날짜 자동 조회
- 기간 선택 후 `조회` 클릭 시 재조회
- 조회 중엔 버튼 비활성
- 동일 조건 재조회는 캐시 우선

## 5) 상태 UX

- 로딩: 카드 스켈레톤 3~5개
- 빈값: "해당 기간 급식 정보가 없습니다"
- 오류: "급식 정보를 불러오지 못했습니다" + 재시도 버튼

## 6) 반응형/접근성 규칙

- 카드 최소 터치 영역 44px 이상
- 메뉴 줄바꿈 유지(가독성 우선)
- 상태 안내는 `aria-live=polite`
- 날짜/버튼은 키보드 조작 가능
