# Phase 8 - 접근성 체크리스트

작성일: 2026-03-26
목표: 키보드/명도/스크린리더 기준으로 주요 화면의 접근성을 확보한다.

## 1) 공통 UI

- [ ] 모든 버튼/링크/입력은 키보드 포커스 가능
- [ ] 포커스 표시가 시각적으로 명확함
- [ ] 포커스 이동 순서가 시각 순서와 일치
- [ ] 다크/라이트 모드 대비 기준 충족

## 2) 검색 화면

- [ ] 시도교육청/학교명 라벨 연결
- [ ] 유효성 오류 메시지가 필드와 연결(`aria-describedby`)
- [ ] 결과 개수/상태 메시지 `aria-live` 제공
- [ ] 결과 카드 액션의 접근 가능한 이름 제공

## 3) 상세/탭

- [ ] 탭 구조(`tablist`, `tab`, `tabpanel`) 준수
- [ ] 방향키 기반 탭 이동 지원
- [ ] 탭 전환 시 스크린리더 상태 인지 가능
- [ ] 탭 오류/빈값 메시지의 의미가 명확함

## 4) 지도 탭

- [ ] 지도 비시각 대체 텍스트/주소 제공
- [ ] 길찾기/거리계산 버튼 라벨 명확
- [ ] 권한 거부 메시지가 스크린리더에 전달됨

## 5) 급식/시간표 탭

- [ ] 표/리스트 헤더와 셀 관계가 명확함
- [ ] 모바일 가로 스크롤 표의 사용 안내 제공
- [ ] 빈 상태에서 다음 액션 문구 제공

## 6) 상태 UI

- [ ] 로딩 상태가 `aria-busy`로 전달됨
- [ ] 오류 상태에 재시도 액션 제공
- [ ] 빈 상태 문구가 기능 맥락을 설명함

## 7) 검사 절차

- [ ] 키보드만으로 전체 플로우 수행
- [ ] 스크린리더(NVDA/VoiceOver 중 1개 이상) 점검
- [ ] Lighthouse Accessibility 점수 기준 설정(예: 90+)
- [ ] 모바일 접근성(터치 영역, 글자 크기) 점검
