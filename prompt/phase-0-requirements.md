# Phase 0 - 요구사항 문서

작성일: 2026-03-26
목표: 학교 정보 조회 서비스의 기능/비기능 요구사항 및 화면 범위를 확정한다.

## 1) 사용자 플로우

1. 사용자가 첫 화면에서 `시도교육청`을 선택한다.
2. 사용자가 `학교명`을 입력하고 조회한다.
3. 시스템이 학교 리스트를 표시한다.
4. 사용자가 학교를 선택하면 상세 페이지로 이동한다.
5. 상세 페이지 상단에서 학교 대표정보를 확인한다.
6. 탭 1에서 학교 상세정보 및 지도(카카오맵)를 확인한다.
7. 탭 2에서 급식식단 정보를 확인한다.
8. 탭 3에서 학년/학급 선택 후 시간표를 확인한다.

## 2) 기능 요구사항

- 검색
- 시도교육청 선택은 필수다.
- 학교명 입력은 필수이며 공백-only 입력은 허용하지 않는다.
- 조회 결과는 리스트 형태로 표시한다.
- 각 리스트 항목은 최소 `학교명, 학교급, 주소`를 표시한다.

- 상세
- 학교 선택 시 상세 페이지로 이동한다.
- 상세 상단은 대표정보(`학교명, 주소, 연락처, 홈페이지`)를 표시한다.
- 상세 하단은 3개 탭으로 구성한다.

- 탭 1: 학교정보/위치
- 학교 기본정보를 텍스트로 표시한다.
- 주소 기반으로 카카오맵을 렌더링하고 마커를 표시한다.
- 길찾기 및 거리계산 기능을 추가한다.

- 탭 2: 급식식단
- 기본 조회 기준은 `오늘 날짜`다.
- 식단이 없는 날짜는 명시적으로 안내한다.

- 탭 3: 시간표
- 학년/학급 선택 UI를 제공한다.
- 선택 완료 후 해당 조건으로 시간표를 조회해 표 형태로 표시한다.

- 즐겨찾기/최근 조회
- 사용자가 학교를 즐겨찾기에 추가/해제할 수 있어야 한다.
- 사용자가 최근 조회한 학교 목록을 확인할 수 있어야 한다.

- 지도 길찾기/거리 계산
- 학교 위치 기준으로 길찾기 실행(외부 지도 연동 포함 가능) 기능을 제공한다.
- 기준 위치(예: 현재 위치)에서 학교까지의 거리 정보를 확인할 수 있어야 한다.

- 테마
- 다크모드 토글 버튼을 제공한다.
- 라이트/다크 모드 전환 시 즉시 반영한다.
- 사용자 선택 테마는 로컬 저장소에 보존한다.

## 3) 비기능 요구사항

- 성능
- 검색 결과 체감 응답은 2초 내를 목표로 한다(일반 네트워크 기준).
- 탭 전환 시 불필요한 재조회는 최소화한다.

- 접근성
- 모든 입력/버튼/탭은 키보드 조작 가능해야 한다.
- 다크/라이트 모두 텍스트 대비를 확보한다.

- 반응형
- 모바일(360px+)~데스크톱에서 레이아웃이 깨지지 않아야 한다.
- 지도/리스트/표는 모바일에서 세로 스크롤로 접근 가능해야 한다.

- 안정성
- API 실패, 빈 데이터, 파라미터 오류에 대해 사용자 안내 메시지를 제공한다.

## 4) 화면 범위 확정

- `/` (첫 화면)
- 시도교육청 선택
- 학교명 입력
- 검색 버튼
- 결과 리스트
- 다크모드 토글

- `/school/[schoolKey]` (상세 화면)
- 대표정보 헤더
- 탭 네비게이션(학교정보/위치, 급식, 시간표)
- 탭 콘텐츠 영역
- 다크모드 토글(공통 헤더 또는 페이지 우상단)

## 5) 제외 범위 (Phase 0 기준)

- 로그인/권한 관리
- 고급 필터(설립유형, 학교급 다중 필터 등)
