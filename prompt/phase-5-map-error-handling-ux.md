# Phase 5 - 지도 오류 처리 UX 문서

작성일: 2026-03-26
목표: 지도 탭의 실패 케이스를 사용자 친화적으로 처리하고 복구 경로를 제공한다.

## 1) 오류 분류

- E1. SDK 로드 실패
- E2. 학교 좌표 없음
- E3. 주소 지오코딩 실패
- E4. 위치 권한 거부
- E5. 거리 계산 실패(위치 획득 timeout 등)
- E6. 외부 길찾기 링크 오픈 실패(팝업 차단)

## 2) 오류별 UX 규칙

### E1. SDK 로드 실패

- 메시지: "지도를 불러오지 못했습니다. 다시 시도해 주세요."
- 액션: `다시 시도` 버튼
- 보조: 학교 주소 텍스트는 계속 노출

### E2. 학교 좌표 없음

- 메시지: "학교 위치 좌표가 없어 지도 표시가 제한됩니다."
- 액션: `주소로 길찾기` 버튼
- 보조: 거리 계산 비활성

### E3. 주소 지오코딩 실패

- 메시지: "주소 기반 위치를 찾지 못했습니다."
- 액션: `주소 복사`, `다시 시도`
- 보조: 주소 원문 노출

### E4. 위치 권한 거부

- 메시지: "거리 계산을 위해 위치 권한이 필요합니다."
- 액션: `권한 안내 보기` (브라우저 설정 안내)
- 보조: 길찾기 기능은 정상 제공

### E5. 거리 계산 실패

- 메시지: "거리 계산에 실패했습니다. 잠시 후 다시 시도해 주세요."
- 액션: `다시 계산`
- 보조: 실패 횟수 누적 시 안내 강화

### E6. 길찾기 링크 오픈 실패

- 메시지: "길찾기 페이지를 열지 못했습니다. 팝업 차단을 확인해 주세요."
- 액션: `링크 복사`

## 3) 상태 컴포넌트 매핑

- 치명적(E1) -> `ErrorState` + Retry
- 비치명적(E2/E3/E4/E5/E6) -> 인라인 경고 배너 + 대체 액션

## 4) 모바일 UX 규칙

- 오류 메시지는 지도 상단 고정 배너로 표시
- 주요 액션 버튼 1개를 우선 배치, 나머지는 보조 링크 처리
- 길찾기 버튼은 하단 고정 CTA 영역에서도 제공 가능

## 5) 접근성 규칙

- 오류 메시지는 `aria-live=polite`
- 오류 발생 시 포커스를 메시지 영역으로 이동 가능하게 구성
- 버튼 라벨은 동사 중심으로 명확화(예: `다시 시도`, `주소 복사`)

## 6) 로그/모니터링 항목

- `map_sdk_load_failed`
- `map_geocode_failed`
- `map_geolocation_denied`
- `map_distance_calc_failed`
- `map_route_open_failed`

## 7) 완료 기준

- 오류별 사용자 메시지 + 최소 1개 복구 액션 제공
- 지도가 실패해도 학교 기본정보 접근 가능
- 모바일/데스크톱 모두에서 메시지 레이아웃이 깨지지 않음
