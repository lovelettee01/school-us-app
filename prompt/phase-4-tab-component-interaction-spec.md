# Phase 4 - 탭 컴포넌트 인터랙션 명세

작성일: 2026-03-26
목표: 상세 페이지 탭(학교정보/위치, 급식, 시간표)의 전환/조회/캐시 동작을 정의한다.

## 1) 탭 구성

- `info` : 학교정보 + 지도 + 길찾기/거리계산
- `meal` : 급식식단
- `timetable` : 시간표

기본 탭: `info`
URL 동기화: `?tab=info|meal|timetable`

## 2) 탭 전환 규칙

1. 사용자가 탭 버튼 클릭
2. `activeTab` 업데이트
3. URL 쿼리 동기화
4. 탭 데이터 캐시 확인
5. 캐시 없으면 조회, 캐시 있으면 즉시 렌더

## 3) 탭별 최초 진입 동작

- `info`
- 대표정보가 있으면 즉시 렌더
- 지도 SDK 로딩 후 지도 출력
- 위치 권한 동의 시 거리 계산 활성화

- `meal`
- 기본 날짜(오늘) 기준 조회
- 데이터 없으면 Empty

- `timetable`
- 초기 상태는 선택 유도(학년/학급 미선택)
- 조건 완성 후 조회 실행

## 4) 탭 상태 유지/초기화 규칙

- 유지 대상
- `activeTab`
- 각 탭의 마지막 조회 파라미터
- 마지막 조회 결과 캐시

- 초기화 대상
- `schoolKey`가 바뀌면 탭 파라미터/결과 전부 초기화
- 상세 페이지를 완전히 이탈하면 메모리 상태 초기화(로컬 저장 제외)

## 5) 재조회 정책

- `info`
- 대표정보는 상세 공통 데이터 재사용
- 지도만 재렌더 필요 시 최소 범위 업데이트

- `meal`
- 날짜/기간 변경 시 재조회
- 동일 파라미터는 캐시 우선

- `timetable`
- 학년/학급/날짜 변경 시 재조회
- 동일 파라미터는 캐시 우선

## 6) 오류/빈값 처리

- 탭 오류는 해당 탭에만 표시
- 탭 오류가 나도 다른 탭 이동 가능
- 탭 빈값은 안내 문구 + 조건 변경 액션 제공

## 7) 접근성 규칙

- 탭은 `role=tablist`, 버튼은 `role=tab`
- 좌우 방향키로 탭 이동 가능
- 활성 탭에 `aria-selected=true`
- 탭 콘텐츠 영역은 `role=tabpanel` 연결

## 8) 성능/UX 규칙

- 탭 전환 체감 속도 우선: 스켈레톤 150ms 지연 표출(짧은 깜빡임 방지)
- 연속 탭 전환 시 중복 요청 취소
- 탭 콘텐츠는 최소 높이를 확보해 레이아웃 점프 방지
