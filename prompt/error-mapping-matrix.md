# 에러 매핑 매트릭스

## 문서 목적
- 기존 에러 노출 방식을 신규 메시지 시스템(`toast/popup/auto`)으로 전환하기 위한 기준표다.

## 진행 체크리스트
- [x] 주요 에러 발생 지점 식별
- [x] 기존 노출 방식 분류(인라인/패널/토스트)
- [x] 신규 메시지 모드 매핑
- [x] 실제 코드 마이그레이션 반영

| 영역 | 기존 방식 | 신규 모드 | 비고 |
|---|---|---|---|
| 검색 입력 검증 실패 | 인라인(`InlineFieldError`) + 에러 상태 | `toast` | 입력 흐름 방해 최소화 |
| 학교 검색 API 실패 | 에러 패널 + 토스트 | `popup`(기본) + 재시도 버튼 유지 | 중요 오류 가시성 강화 |
| 학교 상세 로딩 실패 | 에러 패널 + 토스트 | `popup` | 진입 불가 오류 |
| 급식 조회 실패 | 에러 패널 + 토스트 | `popup` | 조회 실패 원인 전달 |
| 시간표 조회 실패 | 에러 패널 + 토스트 | `popup` | 조회 실패 원인 전달 |
| 지도 SDK/위치 권한 오류 | 토스트 | `toast` | 맥락 유지 필요 |
| 길찾기 링크 생성 실패 | 토스트 | `toast` | 즉시 피드백 중심 |
