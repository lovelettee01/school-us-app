# Phase 4 - 상세 페이지 정보 구조도

작성일: 2026-03-26
목표: 학교 상세 페이지(`/school/[schoolKey]`)의 정보 계층과 섹션 구성을 확정한다.

## 1) 페이지 정보 구조(IA)

```text
SchoolDetailPage
├─ Global Header
│  ├─ Home 이동
│  └─ Theme Toggle
├─ School Hero (대표정보)
│  ├─ 학교명 / 학교급 / 시도교육청
│  ├─ 주소(도로명/지번)
│  ├─ 연락처 / 홈페이지
│  └─ 액션: 즐겨찾기 토글, 최근조회 기록
├─ Quick Actions
│  ├─ 지도 길찾기
│  └─ 거리 계산(현재 위치 기준)
├─ Tabs
│  ├─ Tab 1: 학교정보/위치
│  │  ├─ 학교 기본 상세 정보
│  │  └─ Kakao Map + 마커 + 길찾기/거리 패널
│  ├─ Tab 2: 급식
│  │  ├─ 날짜(또는 기간) 선택
│  │  └─ 급식 카드/리스트
│  └─ Tab 3: 시간표
│     ├─ 학년/학급/날짜 선택
│     └─ 시간표 테이블
└─ 상태/피드백 영역
   ├─ Loading
   ├─ Empty
   └─ Error + Retry
```

## 2) 대표정보(헤더) 필드 확정

- 필수
- 학교명(`schoolName`)
- 시도교육청명(`officeName`)
- 학교급(`schoolType`)
- 도로명 주소(`addressRoad`)

- 선택(있을 때만 표시)
- 지번 주소(`addressJibun`)
- 전화번호(`tel`)
- 홈페이지(`homepage`)
- 설립구분(`orgType`)
- 남녀공학 구분(`coeduType`)

## 3) 데이터 계층

- 레벨 A: 상세 공통 데이터
- 학교 대표정보 + 좌표/주소 기반 정보
- 상세 진입 시 1회 조회

- 레벨 B: 탭별 데이터
- 탭 1: 지도 렌더 및 위치 관련 데이터
- 탭 2: 급식 데이터
- 탭 3: 시간표 데이터

- 레벨 C: 사용자 상태
- 즐겨찾기 여부
- 최근조회 기록
- 활성 탭

## 4) 레이아웃 규칙

- 데스크톱
- 대표정보 헤더 + 우측 액션(즐겨찾기, 길찾기)
- 탭 콘텐츠는 1열이 기본, 정보/지도는 2분할 가능

- 모바일
- 대표정보를 카드형 세로 스택으로 압축
- 탭 고정 헤더(스크롤 시 상단 유지)
- 지도 높이 최소 260px 보장

## 5) 상태 우선순위

1. 상세 공통 조회 실패 시 탭 콘텐츠 렌더 중단 + 오류 상태
2. 공통 성공 후 탭별 오류는 해당 탭 내부에서만 처리
3. 탭 데이터 없음은 Empty로 처리하고 다른 탭은 정상 이용 가능
