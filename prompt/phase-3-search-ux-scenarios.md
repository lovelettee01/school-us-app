# Phase 3 - 검색 UX 시나리오 문서

작성일: 2026-03-26
목표: 사용자의 검색 행동, 예외 상황, 결과 탐색 흐름을 시나리오로 정의한다.

## 1) 기본 시나리오

1. 사용자가 홈(`/`)에 진입한다.
2. 시도교육청을 선택한다.
3. 학교명을 입력한다.
4. `조회` 버튼 클릭 또는 엔터 입력으로 검색한다.
5. 로딩 상태를 본다.
6. 검색 결과 리스트를 확인한다.
7. 원하는 학교의 `상세 보기`를 선택한다.
8. 상세(`/school/[schoolKey]`)로 이동한다.

## 2) 입력 검증 시나리오

- 시도교육청 미선택 + 조회
- 인라인 오류: "시도교육청을 선택해 주세요."

- 학교명 미입력/공백 + 조회
- 인라인 오류: "학교명을 입력해 주세요."

- 1글자 입력 + 조회
- 인라인 오류: "학교명은 2글자 이상 입력해 주세요."

## 3) 결과 상태 시나리오

- 결과 1건 이상
- 총 건수 노출 + 카드 리스트 노출
- 카드 클릭 또는 상세 버튼으로 상세 이동

- 결과 0건
- 빈 상태 메시지 + "검색 조건 다시 설정" 가이드

- API 오류
- 오류 상태 메시지 + 재시도 버튼
- 재시도 시 직전 파라미터로 재조회

## 4) 최근 조회/즐겨찾기 시나리오

- 최근 조회
- 상세 이동 성공 시 해당 학교를 최근 조회 목록 상단에 추가
- 중복 항목은 최신 시점으로 갱신

- 즐겨찾기
- 검색 결과 카드에서 즐겨찾기 토글 가능
- 토글 즉시 UI 반영
- 로컬 저장소 반영 실패 시 안내 토스트 노출

## 5) 리스트 탐색 시나리오

- 기본 정렬: 학교명 오름차순
- 결과가 많을 경우
- 우선 `더보기` 방식 적용(모바일 친화)
- 필요 시 페이지네이션 전환 가능

## 6) 접근성 시나리오

- 탭 키로 입력 -> 버튼 -> 결과 카드 순서 탐색 가능
- 엔터로 조회, 스페이스/엔터로 카드 액션 실행 가능
- 상태 메시지(로딩 완료, 결과 개수)는 `aria-live`로 전달

## 7) 성능 시나리오

- 동일 검색 조건 재실행 시 캐시 활용
- 연속 조회 중 이전 요청 취소(경합 방지)
- 2초 이상 지연 시 보조 안내 문구 노출

## 8) 실패/복구 시나리오

- 네트워크 단절
- 오류 메시지 + 재시도
- 연결 복구 후 재시도 성공 시 정상 리스트 복귀

- 상세 이동 실패(잘못된 키)
- 상세 페이지에서 오류 상태 표시
- 홈으로 돌아가기 액션 제공
