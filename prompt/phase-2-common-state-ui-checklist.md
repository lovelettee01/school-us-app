# Phase 2 - 공통 상태 UI 컴포넌트 체크리스트

작성일: 2026-03-26
목표: 검색/상세/탭에서 재사용할 상태 UI 컴포넌트를 정의하고 검수 기준을 만든다.

## 1) 대상 컴포넌트

- `LoadingState`
- `ErrorState`
- `EmptyState`
- `InlineFieldError`
- `RetryButton`

## 2) 공통 요구사항 체크리스트

- [ ] Props 인터페이스가 단순하고 일관적인가
- [ ] 라이트/다크 테마에서 가독성이 유지되는가
- [ ] 모바일에서 레이아웃이 깨지지 않는가
- [ ] 접근성 속성(`role`, `aria-live`)이 적용되는가
- [ ] 텍스트 메시지가 화면별로 커스터마이즈 가능한가

## 3) 상태별 컴포넌트 기준

### LoadingState

- [ ] 스켈레톤 또는 스피너를 제공한다
- [ ] 조회 영역 크기와 유사한 플레이스홀더를 제공한다
- [ ] 중복 클릭 방지를 위해 액션 버튼 비활성화를 지원한다
- [ ] `aria-busy=true`를 상위 컨테이너에서 설정 가능하다

### ErrorState

- [ ] 기본 제목 + 상세 메시지 표시 가능
- [ ] 선택적 `재시도` 버튼 제공
- [ ] 오류 코드 표시(개발 모드 한정) 옵션 제공
- [ ] 네트워크/상류 API 오류 문구를 구분할 수 있다

### EmptyState

- [ ] "결과 없음" 기본 문구 제공
- [ ] 상황별 안내(조건 변경, 날짜 변경) 메시지 제공
- [ ] 선택적 보조 액션 버튼 제공

### InlineFieldError

- [ ] 입력 필드와 연계 가능한 id/for 구성을 지원
- [ ] 검증 실패 즉시 메시지 노출 가능
- [ ] 스크린리더 읽기 순서가 자연스러운가

## 4) 화면 매핑 체크

- 홈 검색
- [ ] 입력 검증 오류 -> `InlineFieldError`
- [ ] 검색 중 -> `LoadingState`
- [ ] 결과 없음 -> `EmptyState`
- [ ] API 오류 -> `ErrorState`

- 상세 공통
- [ ] 대표정보 로딩 -> `LoadingState`
- [ ] 상세 실패 -> `ErrorState`

- 탭 1(학교정보/지도)
- [ ] 좌표 없음/지도 불가 -> `EmptyState` 또는 `ErrorState`
- [ ] 위치권한 거부 -> 안내 배너 + 거리 계산 비활성

- 탭 2(급식)
- [ ] 급식 없음 -> `EmptyState`
- [ ] 조회 실패 -> `ErrorState`

- 탭 3(시간표)
- [ ] 학년/학급 미선택 -> 안내형 `EmptyState`
- [ ] 조회 실패 -> `ErrorState`

## 5) 메시지 카피 가이드

- 사용자 액션 유도형 문장 사용
- 예: "조회 조건을 바꿔 다시 시도해 주세요."
- 기술적 원인 노출 최소화(운영환경)

## 6) 완료 기준(Definition of Done)

- [ ] 상태 컴포넌트가 최소 3개 화면에서 재사용된다
- [ ] 모든 상태에서 키보드 접근 및 스크린리더 동작이 검증된다
- [ ] 다크/라이트 모드 시각 검수가 완료된다
- [ ] 재시도 동작이 실제 API 호출과 연결되어 검증된다
